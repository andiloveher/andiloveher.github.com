HTTP版本
  • 0.9 采用纯文本格式，且只读 get
  • 1.0 加入post、head等方法，协议版本号，文件处理类型，http header， 状态码
  • 1.1 加入put、delete等方法，持久连接，强制要求Host头，引入更多的缓存机制
  • 2 域名分片/多路复用 解决头部阻塞，二进制分帧，头部压缩 使用首部表来跟踪和存储键值对，服务端推送
  • 3 基于udp
HTTP特点
  • 无状态
  • 明文传输
  • 队头阻塞 原因开启长链接，公用同一个tcp
Header
  • Content-Type 数据格式 表单 form-data / x-www-form-urlencoded
  • Content-Encoding 压缩方式
  • Content-Length / Transfer-Encoding 内容长度
cookie
  • 作用域 Path / Domain
  • 时间 Expires / Max-Age
  • 安全 HttpOnly / SameSite / Secure
强制缓存、协商缓存
  • Cache-Control 验证强缓存是否可用
  • If-Modified-Since / If-None-Match
正向、反向代理
  • 负载均衡
  • 代理缓存
  • X-Forward-For / X-Real-IP
HTTPS
  • 对称加密、非对称加密
  • 握手过程
  • 签名验证 ca数字证书保证服务器身份，防止dns篡改
  1. client 发送 client_random和加密方法列表
  2. server 返回 server_random、加密方法和公钥
  3. client生成一个pre_random，利用公钥加密，传给服务器
  4. server利用私钥解密，获取pre_random
  5. 利用对称加密混合三个随机数生成密钥，进行通信
HTTP2
  • 流 已建立的连接上的双向字节流
  • 消息 一系列的数据帧
  • 通信的最小单位，包含帧首部

http2.0通信都在一个连接上完成，这个连接可以承载任意数据量的双向数据流。每个数据流以消息的形式发送，消息由一个或多个帧组成。所有的帧都采用二进制编码，所有的首部数据都会被压缩。

  • 多路复用
  • 请求优先级
  • 服务端推送
  • 首部压缩
TCP和UDP
  • TCP面向连接，可靠的，基于字节流的传输层协议；UDP面向无连接的基于数据报的传输层协议；
  • TCP的三次握手和四次挥手
  • 流量控制 滑动窗口
  • 拥塞窗口 发送窗口控制
  • TFO syn cookie
跨域
  • 同源策略
  • cors / jsonp / nginx
  • 简单请求 / 非简单请求

  • XSS 跨站脚本攻击

    • 获取cookie等隐私
